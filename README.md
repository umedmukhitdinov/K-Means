# K-Means Visualizer

Интерактивный учебный проект, который показывает, как работает алгоритм кластеризации **K-Means** шаг за шагом.

Если вы новичок, это приложение можно воспринимать так:
- есть много случайных точек на поле;
- мы выбираем число групп `K` (например, 3);
- алгоритм пытается «разложить» точки по этим группам;
- вы видите каждый шаг глазами: где центры групп, как они двигаются, как уменьшается ошибка.

---

## Что это за проект

**K-Means Visualizer** — это веб-приложение на React, в котором можно:
- генерировать набор точек;
- запускать алгоритм автоматом или по шагам;
- смотреть фазу алгоритма (E-шаг / M-шаг);
- видеть график ошибки `SSE` по итерациям;
- читать краткую теорию прямо внутри интерфейса.

Проект полезен для:
- студентов и начинающих в ML;
- объяснения K-Means на уроках/презентациях;
- быстрого визуального понимания, что происходит «под капотом».

---

## Простое объяснение K-Means

Представьте, что у вас есть много точек на карте, и вы хотите разделить их на несколько «районов».

Алгоритм делает это так:

1. Выбирает `K` начальных центров (центроидов).
2. Для каждой точки ищет ближайший центр.
3. Перемещает каждый центр в середину своих точек.
4. Повторяет шаги 2–3, пока центры почти не перестанут двигаться.

Когда центры стабилизировались — алгоритм сошёлся.

---

## Основные функции приложения

- Настройка количества кластеров `K`.
- Настройка количества точек.
- Регулировка скорости анимации.
- Кнопки:
  - `Generate Points` — создать новый набор точек;
  - `Start` / `Pause` — запустить или остановить;
  - `Step` — сделать один следующий шаг вручную;
  - `Reset` — сбросить текущее состояние.
- Опция показа расстояний от точек до центроидов.
- Визуализация перемещения центроидов стрелками.
- График `SSE` (функции потерь), который обычно уменьшается по мере итераций.

## Установка и запуск (для новичка)

### 1) Что нужно установить

Установите:
- **Node.js LTS** (вместе с npm): [https://nodejs.org](https://nodejs.org)

Проверьте в терминале:

```bash
node -v
npm -v
```

Если команды показывают версии — всё готово.

### 2) Установка зависимостей

Откройте терминал в папке проекта и выполните:

```bash
npm install
```

### 3) Запуск в режиме разработки

```bash
npm run dev
```

После запуска в терминале появится локальный адрес (обычно `http://localhost:5173`).
Откройте его в браузере.

### 4) Сборка production-версии

```bash
npm run build
```

### 5) Локальный просмотр production-сборки

```bash
npm run preview
```

## Как пользоваться приложением пошагово

1. Нажмите **Generate Points** — появятся точки.
2. Выберите `K` (например, 3 или 4).
3. Нажмите **Start**.
4. Наблюдайте:
   - как точки раскрашиваются по кластерам;
   - как центроиды двигаются;
   - как меняется `SSE`.
5. Для подробного разбора используйте **Pause** и **Step**.
6. Когда увидите статус **Converged**, алгоритм завершил работу.

---

## Что означают фазы

- **Idle** — ожидание, алгоритм не запущен.
- **Assign (E-шаг)** — точки назначаются ближайшему центроиду.
- **Update (M-шаг)** — центроиды пересчитываются как среднее своих точек.
- **Converged** — центроиды почти не меняются, процесс завершён.

## Частые проблемы и решения

### `npm` не найден

Причина: Node.js не установлен или не добавился в PATH.
Решение: переустановите Node.js LTS с официального сайта и перезапустите терминал.

### Порт занят (например, 5173)

Vite обычно сам предложит другой порт. Просто откройте тот адрес, который выведен в терминале.

### После изменений ничего не видно

Проверьте ошибки в терминале и в консоли браузера (F12).
